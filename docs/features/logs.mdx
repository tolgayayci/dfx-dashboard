## Log Data Structure

Understanding the log data structure helps with effective debugging and analysis:

### Command History Format

<AccordionGroup>
  <Accordion title="Log Entry Structure">
    Each command history entry contains:
    - **Timestamp**: Date and time of execution
    - **Command**: Full dfx command with parameters
    - **Path**: Working directory context
    - **Result**: Complete command output
    - **Subcommand**: Command category for filtering
  </Accordion>
  
  <Accordion title="Parsing Logic">
    - Commands are parsed from log files automatically
    - Timestamps are extracted and formatted for display
    - Results are cleaned and formatted for readability
    - Paths are extracted for context information
  </Accordion>
</AccordionGroup>

### Canister Log Format

<AccordionGroup>
  <Accordion title="Log Message Structure">
    Canister logs follow the format:
    ```
    [count. YYYY-MM-DDTHH:mm:ss.sssZ]: message
    ```
    - **Count**: Sequential log entry number
    - **ISO Timestamp**: Precise timestamp in ISO format
    - **Message**: The actual log content
  </Accordion>
  
  <Accordion title="Message Processing">
    - Automatic parsing of timestamp components
    - Message content preserved exactly as logged
    - Empty lines and malformed entries handled gracefully
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Command History Issues">
    **Common command history problems:**
    
    **Problem**: Commands not appearing in history
    - Ensure commands are executed through the DFX Dashboard
    - Check if the command log file is accessible
    - Verify the command completed execution
    - Restart the dashboard if logs aren't updating
    
    **Problem**: Command output is incomplete
    - Check if the command is still running
    - Verify network connectivity for network commands
    - Look for error messages in the output modal
    - Try re-executing the command for complete output
  </Accordion>
  
  <Accordion title="Canister Log Issues">
    **Common canister log problems:**
    
    **Problem**: No logs appearing for canister
    - Verify the canister is deployed and running
    - Check network connectivity to the canister
    - Ensure the canister has generated log entries
    - Try refreshing by selecting the canister again
    
    **Problem**: Canister not appearing in dropdown
    - Verify the project is loaded in the dashboard
    - Check if the canister is properly deployed
    - Ensure dfx.json configuration is correct
    - Try refreshing the projects list
  </Accordion>
  
  <Accordion title="Search and Filter Issues">
    **Common search and filter problems:**
    
    **Problem**: Search not finding expected results
    - Check search term spelling and case
    - Verify the logs contain the expected content
    - Try broader search terms
    - Clear search and try again
    
    **Problem**: Filters not working correctly
    - Ensure the filter criteria match available data
    - Try resetting filters to default
    - Check if data is loading properly
    - Refresh the page if filters seem stuck
  </Accordion>
</AccordionGroup>
