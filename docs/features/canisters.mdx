The Canisters page in the DFX Dashboard provides a centralized interface for viewing and managing all your canisters across different projects, including both user canisters and Network Nervous System (NNS) canisters. This powerful tool allows you to interact with and monitor your Internet Computer canisters efficiently with full dfx 0.25.0 compatibility.

## Canister Types

The dashboard supports two types of canisters:

<Tabs>
  <Tab title="User Canisters">
    ### User Canisters
    
    <CardGroup cols={1}>
      <Card title="Your Deployed Canisters" icon="user">
        - Canisters deployed by you in your projects
        - Full management capabilities available
        - Complete control over lifecycle and configuration
        - Interactive command interface for all operations
      </Card>
    </CardGroup>
  </Tab>
  
  <Tab title="NNS Canisters">
    ### NNS Canisters
    
    <CardGroup cols={1}>
      <Card title="Network Nervous System" icon="network-wired">
        - Network Nervous System canisters (governance, ledger, etc.)
        - Special NNS badges and indicators
        - Query-only operations for safety
        - Available on both local and IC networks
      </Card>
    </CardGroup>
  </Tab>
</Tabs>

## Advanced Filtering

Efficiently manage large numbers of canisters with powerful filtering:

<AccordionGroup>
  <Accordion title="Search Functionality">
    **Real-time search** by canister name or canister ID - results update as you type
  </Accordion>
  
  <Accordion title="Filter by Type">
    - **All Canisters**: Show both user and NNS canisters
    - **User Canisters**: Show only your deployed canisters
    - **NNS Canisters**: Show only Network Nervous System canisters
  </Accordion>
  
  <Accordion title="Filter by Network">
    - **All Networks**: Show canisters from all networks
    - **Local**: Canisters on your local dfx network
    - **IC**: Canisters on the Internet Computer mainnet
  </Accordion>
</AccordionGroup>

## Canister List Table

The table displays essential canister information:

<Steps>
  <Step title="Canister Name">
    Canister name with NNS badge for Network Nervous System canisters
  </Step>
  <Step title="Type">
    "User", "Network Nervous System", or other canister types
  </Step>
  <Step title="Project Name">
    Project association (clickable for user canisters, "Network Nervous System" for NNS)
  </Step>
  <Step title="Action">
    "Canister Actions" button leading to the detail page
  </Step>
</Steps>

<Tip>
Click column headers to sort canisters by name, type, or project for easier management across multiple networks.
</Tip>

## Canister Detail Pages

Click any canister to access comprehensive management tools with two main tabs:

<Tabs>
  <Tab title="User Canisters">
    ### User Canister Detail View
    
    <img
      src="/images/features/canisters/user-canister-detail.png"
      alt="User Canister Detail"
      className="rounded-lg border"
    />
    
    **Interface:**
    - **Methods Tab**: Interactive command interface for canister operations
    - **Metadata Tab**: Comprehensive canister metadata and configuration
    
    **Management Actions:**
    - Top Up, Status, Config, Remove buttons in header
    - Full control over canister lifecycle
  </Tab>
  
  <Tab title="NNS Canisters">
    ### NNS Canister Detail View
    
    <img
      src="/images/features/canisters/nns-canister-detail.png"
      alt="NNS Canister Detail"
      className="rounded-lg border"
    />
    
    **Interface:**
    - **Methods Tab**: Query-only operations for NNS interactions
    - **Metadata Tab**: System canister metadata and network information
    
    **Read-Only Access:**
    - No action buttons (safety for system canisters)
    - Query operations only
  </Tab>
</Tabs>

## Canister Actions

Comprehensive management capabilities for user canisters:

<CardGroup cols={2}>
  <Card title="Top Up" icon="coins">
    Add cycles to keep the canister running
  </Card>
  <Card title="Status" icon="chart-line">
    View current status, cycle balance, and health metrics
  </Card>
  <Card title="Config" icon="gear">
    View and edit canister configuration
  </Card>
  <Card title="Remove" icon="trash">
    Delete canister (irreversible - use with caution)
  </Card>
</CardGroup>

<Warning>
The "Remove" action permanently deletes the canister and all associated data. Always ensure you have backups before proceeding.
</Warning>

## Command Interface (dfx 0.25.0)

The Methods tab provides direct access to dfx commands with full 0.25.0 compatibility:

### Key Features

<CardGroup cols={2}>
  <Card title="Updated Commands" icon="terminal">
    All dfx 0.25.0 commands with latest options
  </Card>
  <Card title="Dynamic Interface" icon="sliders">
    Command-specific options appear automatically
  </Card>
  <Card title="Direct Execution" icon="play">
    Execute commands directly from the dashboard
  </Card>
  <Card title="Real-time Output" icon="file-text">
    View command results immediately
  </Card>
</CardGroup>

### Available Commands

<CodeGroup>
```bash Core Operations
dfx canister call
dfx canister create
dfx canister delete
dfx canister status
dfx canister install
```

```bash Management
dfx canister deposit-cycles
dfx canister metadata
dfx canister update-settings
dfx canister logs
dfx canister url
```
</CodeGroup>

- **Errors**: Detailed error messages if command fails

## Best Practices

<AccordionGroup>
  <Accordion title="Efficient Management">
    - Use filtering to quickly locate specific canisters
    - Monitor cycle balances regularly
    - Use descriptive naming conventions
  </Accordion>
  
  <Accordion title="NNS Integration">
    - Leverage NNS canisters for network monitoring
    - Use query interfaces for governance insights
    - Monitor system canister health
  </Accordion>
  
  <Accordion title="Development Workflow">
    - Backup critical data before major changes
    - Test commands in development environments first
    - Keep dfx version at 0.25.0 or newer
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Canister Issues">
    **Not responding**: Check status, cycles, and network connectivity
    **Command failures**: Review output for error details and verify parameters
  </Accordion>
  
  <Accordion title="Filtering Problems">
    **Can't find canisters**: Clear filters and search terms, verify correct type/network selection
  </Accordion>
  
  <Accordion title="NNS Access">
    **NNS errors**: Ensure query-only operations and verify network connectivity
  </Accordion>
</AccordionGroup>
