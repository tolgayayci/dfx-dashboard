The Wallet page provides comprehensive management of your Internet Computer wallet, including controller management, address book functionality, custodian operations, and cycle transfers. This page centralizes all wallet-related operations in a professional tabbed interface with full dfx 0.25.0 compatibility.

## Overview

<img
  src="/images/features/wallet/wallet-overview.png"
  alt="Wallet Overview"
  className="rounded-lg border"
/>

The Wallet page is organized into six main functional areas, each designed to handle specific aspects of wallet management:

<CardGroup cols={3}>
  <Card title="Controllers" icon="shield">
    Manage wallet controllers with administrative access
  </Card>
  <Card title="Address Book" icon="address-book">
    Store and manage addresses locally with DFX integration
  </Card>
  <Card title="Custodians" icon="users">
    Authorize and manage custodians with limited permissions
  </Card>
  <Card title="Cycle Transfers" icon="arrow-right-arrow-left">
    Send cycles between canisters and wallets
  </Card>
  <Card title="Wallet Settings" icon="gear">
    Configure wallet name and perform upgrades
  </Card>
  <Card title="Faucet" icon="droplet">
    Redeem faucet coupons for development cycles
  </Card>
</CardGroup>

<Tip>
The wallet functionality requires an active wallet canister. Ensure you have a wallet set up before using these features. You can create a wallet using the `dfx identity deploy-wallet` command.
</Tip>

## Wallet Management Tabs

<Tabs>
  <Tab title="Controllers">
    ## Controllers Tab
    
    <img
      src="/images/features/wallet/controllers-tab.png"
      alt="Controllers Management"
      className="rounded-lg border"
    />
    
    Controllers have administrative access to your wallet canister. This tab provides comprehensive controller management capabilities:
    
    ### Controller Overview
    
    <CardGroup cols={2}>
      <Card title="View Controllers" icon="eye">
        List all current controllers with their principals and copy functionality
      </Card>
      <Card title="Controller Status" icon="chart-line">
        Display controller count and authorization status
      </Card>
    </CardGroup>
    
    ### Adding Controllers
    
    <Steps>
      <Step title="Initiate Addition">
        Click the **"Add Controller"** button to start the process
      </Step>
      <Step title="Enter Principal">
        Enter the **principal ID** of the new controller in the input field
      </Step>
      <Step title="Select Network">
        Choose the appropriate **network** (local or IC) for the operation
      </Step>
      <Step title="Authorize">
        Click **"Add"** to authorize the new controller with administrative access
      </Step>
    </Steps>
    
    ### Removing Controllers
    
    <Steps>
      <Step title="Locate Controller">
        Find the controller you want to remove from the list
      </Step>
      <Step title="Initiate Removal">
        Click the **trash icon** next to their principal
      </Step>
      <Step title="Confirm Action">
        Confirm the removal in the confirmation dialog
      </Step>
      <Step title="Complete Removal">
        The controller will be removed from your wallet
      </Step>
    </Steps>
    
    <Warning>
    Be very careful when managing controllers. Removing all controllers can make your wallet inaccessible. Always ensure you maintain at least one controller with administrative access.
    </Warning>
  </Tab>
  
  <Tab title="Address Book">

    <img
      src="/images/features/wallet/address-book-tab.png"
      alt="Address Book"
      className="rounded-lg border"
    />
    
    The Address Book provides local storage for frequently used addresses and principals with DFX integration:
    
    ### Key Features
    
    <CardGroup cols={2}>
      <Card title="Local Storage" icon="database">
        Addresses are stored locally in the app for quick access
      </Card>
      <Card title="DFX Integration" icon="terminal">
        Also displays addresses from `dfx wallet addresses`
      </Card>
      <Card title="Search & Filter" icon="magnifying-glass">
        Find addresses quickly with advanced filtering
      </Card>
      <Card title="Import/Export" icon="arrow-up-down">
        Manage your address collection with backup capabilities
      </Card>
    </CardGroup>
    
    ### Adding New Addresses
    
    <Steps>
      <Step title="Start Creation">
        Click the **"Add Address"** button to open the form
      </Step>
      <Step title="Fill Details">
        Complete the address information:
        - **Label**: Descriptive name for easy identification
        - **Address**: Principal ID or account identifier
        - **Type**: Select "Principal" or "Account"
        - **Notes**: Optional description for context
      </Step>
      <Step title="Save Address">
        Click **"Save"** to store the address in your local collection
      </Step>
    </Steps>
    
    ### Address Management
    
    <AccordionGroup>
      <Accordion title="Edit Addresses">
        - Click the **edit icon** to modify address details
        - Update labels, notes, or address information
        - Save changes to maintain current information
      </Accordion>
      
      <Accordion title="Delete Addresses">
        - Remove addresses you no longer need
        - Confirm deletion to prevent accidental removal
        - Clean up your address book regularly
      </Accordion>
      
      <Accordion title="Copy & Search">
        - **One-click copying** of addresses to clipboard
        - **Search functionality** to filter by label or address
        - **Quick access** to frequently used addresses
      </Accordion>
    </AccordionGroup>
    
    <Tip>
    Use descriptive labels for your addresses to make them easy to find. Consider organizing by project or purpose (e.g., "Main Canister", "Test Account", "Production Wallet").
    </Tip>
  </Tab>
  
  <Tab title="Custodians">
    ## Custodians Tab
    
    <img
      src="/images/features/wallet/custodians-tab.png"
      alt="Custodians Management"
      className="rounded-lg border"
    />
    
    Custodians can perform operations on behalf of your wallet but have more limited permissions than controllers:
    
    ### Custodian Overview
    
    <CardGroup cols={2}>
      <Card title="View Custodians" icon="users">
        List authorized custodians with their principals and permissions
      </Card>
      <Card title="Authorization Status" icon="shield-check">
        Show authorization status and permission levels
      </Card>
    </CardGroup>
    
    ### Authorizing Custodians
    
    <Steps>
      <Step title="Start Authorization">
        Click the **"Authorize Custodian"** button
      </Step>
      <Step title="Enter Principal">
        Enter the **principal ID** to authorize as custodian
      </Step>
      <Step title="Select Network">
        Choose the appropriate **network** for the operation
      </Step>
      <Step title="Grant Permissions">
        Click **"Authorize"** to grant custodian permissions
      </Step>
    </Steps>
    
    ### Deauthorizing Custodians
    
    <Steps>
      <Step title="Locate Custodian">
        Find the custodian to remove from the authorized list
      </Step>
      <Step title="Initiate Removal">
        Click the **"Deauthorize"** button next to their entry
      </Step>
      <Step title="Confirm Action">
        Confirm the deauthorization action
      </Step>
      <Step title="Complete Removal">
        The custodian will lose wallet permissions
      </Step>
    </Steps>
    
    <Info>
    Custodians can send cycles and perform basic wallet operations but cannot add/remove controllers or other custodians. This provides a secure way to grant limited access.
    </Info>
  </Tab>

    ### Wallet Balance Management
    
    <CardGroup cols={3}>

      </Card>

    <Steps>

    <AccordionGroup>
      <Accordion title="Amount Validation">
        - **Balance Verification**: Ensures sufficient balance before transfer
        - **Minimum Amount**: Validates minimum transfer requirements
        - **Fee Calculation**: Accounts for network fees in validation
      </Accordion>
      
      <Accordion title="Unit Conversion">
        - **Multiple Units**: Support for T (Trillion), G (Billion), M (Million), K (Thousand)
        - **Automatic Conversion**: Seamless conversion between units
        - **Precision Handling**: Accurate handling of large cycle amounts
      </Accordion>
      
      <Accordion title="Transaction Management">
        - **Transaction History**: View recent transfers with details
        - **Error Handling**: Clear feedback for failed operations
        - **Status Tracking**: Monitor transfer completion status
      </Accordion>
    </AccordionGroup>
    
    <Tip>
    Always verify the destination address before sending cycles. Transactions cannot be reversed once confirmed. Use the memo field to track the purpose of transfers.
    </Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Wallet Access Issues">
    **Common wallet access problems:**
    
    **Problem**: Wallet not found or inaccessible
    - Ensure you have a wallet canister deployed
    - Verify you're using the correct identity
    - Check network connectivity and configuration
    
    **Problem**: Permission denied errors
    - Ensure you have appropriate controller/custodian permissions
    - Verify identity matches wallet controller
    - Check network selection matches wallet deployment
  </Accordion>
  
  <Accordion title="Transaction Issues">
    **Common transaction problems:**
    
    **Problem**: Insufficient cycles for operations
    - Check your wallet balance before operations
    - Ensure sufficient cycles for transaction fees
    - Consider topping up wallet from other sources
    
    **Problem**: Transfer failures
    - Verify destination addresses are correct
    - Check network connectivity and status
    - Ensure recipient can receive cycles
  </Accordion>
  
  <Accordion title="Configuration Problems">
    **Common configuration issues:**
    
    **Problem**: Balance not updating
    - Use the refresh button to update manually
    - Check network connectivity
    - Restart the application if issues persist
    
    **Problem**: Upgrade failures
    - Check network status and connectivity
    - Verify sufficient cycles for upgrade
    - Try again after network issues resolve
    
    **Problem**: Coupon redemption fails
    - Verify coupon code is correct and not expired
    - Check network selection matches coupon requirements
    - Ensure wallet can receive cycles
  </Accordion>
</AccordionGroup>

## Security Considerations

<AccordionGroup>
  <Accordion title="Access Control">
    **Critical security practices:**
    - **Controller Access**: Only add trusted principals as controllers
    - **Custodian Permissions**: Regularly review and audit custodian access
    - **Principle of Least Privilege**: Grant minimum necessary permissions
  </Accordion>
  
  <Accordion title="Transaction Security">
    **Safe transaction practices:**
    - **Address Verification**: Always double-check addresses before transfers
    - **Network Selection**: Ensure you're using the correct network for operations
    - **Amount Verification**: Verify transfer amounts before confirmation
  </Accordion>
  
  <Accordion title="Data Protection">
    **Information security measures:**
    - **Backup Strategy**: Maintain secure backups of wallet credentials
    - **Regular Audits**: Periodically review controllers and custodians
    - **Secure Storage**: Store sensitive information in encrypted locations
  </Accordion>
</AccordionGroup>
